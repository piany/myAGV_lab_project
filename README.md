# myAGV Summer School Lab

**ROS2 Humble · Python 3.10+ · Ubuntu 22.04**

A complete 3-phase robotics lab for the Elephant Robotics myAGV platform,
covering SLAM, autonomous navigation, and LLM-powered task planning.

---

## Quick Start (Simulation — no robot needed)

```bash
# 1. Create a virtual environment (required — system Python is externally managed)
python3 -m venv .venv
source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set your DeepSeek API key (required for LLM translation)
export DEEPSEEK_API_KEY="sk-..."
# To persist it across sessions, add the line above to your ~/.bashrc

# 4. Phase 1 — SLAM
python3 -m myagv_lab.phase1_slam.slam_node --sim

# 5. Phase 2 — Navigation (interactive)
python3 -m myagv_lab.phase2_nav.nav_node --sim

# 6. Phase 3 — Full pipeline (no API key, hard-coded PDDL)
python3 -m myagv_lab.phase3_delivery.mission_manager --sim --fallback

# 6b. Phase 3 — With LLM translation (requires DEEPSEEK_API_KEY)
python3 -m myagv_lab.phase3_delivery.mission_manager --sim \
  --task "Deliver package_A to the delivery area and return home."

# 7. Run all tests (PYTHONPATH="" prevents ROS2 plugins from interfering)
PYTHONPATH="" python3 -m pytest tests/test_all_phases.py -v
# Or use the provided script:
./run_tests.sh tests/test_all_phases.py -v
```

## Sim-to-Real Transition

| Step | Action |
|------|--------|
| 1 | `export MYAGV_USE_SIM=0` |
| 2 | Source ROS2: `source /opt/ros/humble/setup.bash` |
| 3 | Build: `cd ~/ros2_ws && colcon build && source install/setup.bash` |
| 4 | Phase 1: `ros2 launch myagv_lab slam_launch.py` |
| 5 | Phase 2: `ros2 launch myagv_lab nav2_launch.py` |
| 6 | Phase 3: `ros2 launch myagv_lab full_demo_launch.py` |

Update `phase2_nav/nav_node.py → WAYPOINTS` with real map coordinates
(read from RViz2 after Phase 1).

## Pipeline (Phase 3)

```
Student types a task in natural language
           │
           ▼  llm_translator.py  (Claude/OpenAI/Deepseek API)
  PDDL domain.pddl + problem.pddl
           │
           ▼  pddl_solver.py  (pyperplan BFS/A*)
  [navigate, load-package, deliver-package, …]
           │
           ▼  primitive_executor.py
  Nav2 goals  +  /cobot_command topics
           │
           ▼
  Physical robot motion
```

## Student Exercises

1. **Trace the pipeline** — add `print()` statements between each stage and observe the data transformations.
2. **Modify the task** — try different natural language descriptions; observe how the PDDL problem changes.
3. **Add a new location** — add `"inspection_area"` to `WAYPOINTS` and the PDDL domain.
4. **Extend the domain** — add a `recharge` action and a `charger_station` waypoint.
5. **Failure handling** — make `navigate()` retry once on failure; add a timeout to the cobot wait loop.
6. **Real deployment** — follow the Sim-to-Real Transition table above.
